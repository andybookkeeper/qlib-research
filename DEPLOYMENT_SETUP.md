# Deployment Setup Guide

This guide walks you through setting up live deployment for the Qlib Trading App on Vercel (frontend) and Render (backend).

## Prerequisites
- GitHub account (already have)
- GitHub CLI or web access to settings

## Quick Start

### 1. Create Vercel Account & Project

**a) Sign up on Vercel**
- Go to https://vercel.com
- Click "Sign Up" and choose "Continue with GitHub"
- Authorize Vercel to access your repositories
- You'll be taken to the dashboard

**b) Create Frontend Project**
- Click "Add New..." → "Project"
- Find and select `andybookkeeper/qlib-research`
- Import the project
- Framework Preset: **Vite**
- Root Directory: **src/app/frontend**
- Build Command: `npm run build` (should auto-fill)
- Output Directory: `dist` (should auto-fill)
- Environment Variables: (leave empty for now)
- Click "Deploy"
- Wait for build to complete (~2-3 minutes)

**c) Get Your Vercel Credentials**
After deployment:
- Note the **Deployment URL** (e.g., `qlib-trading-ui.vercel.app`)
- Go to Account Settings (top right) → Tokens
- Create a new token with name "GitHub CI/CD" → Copy the token
- Go to Project Settings → General
- Find **Project ID** in the URL or settings
- Find **Team ID/Org ID** in Account Settings

### 2. Create Render Account & Service

**a) Sign up on Render**
- Go to https://render.com
- Click "Sign Up" → Choose "GitHub" → Authorize
- Accept the installation prompt

**b) Create Backend Service**
- Dashboard → "New +" → "Web Service"
- Connect your GitHub repo (if not already)
- Select `andybookkeeper/qlib-research`
- Name: `qlib-api`
- Region: **Oregon** (default)
- Branch: **master**
- Runtime: **Python 3**
- Build Command: `pip install -r requirements.txt`
- Start Command: `uvicorn src.qlib_research.app.api.main:app --host 0.0.0.0 --port $PORT`
- Instance Type: **Free** (or Starter for production)
- Environment Variables:
  ```
  FASTAPI_ENV=production
  SECRET_KEY=<randomly generated by Render>
  CORS_ORIGINS=https://qlib-trading-ui.vercel.app
  ENABLE_LIVE_TRADING=false
  PAPER_TRADING_MODE=true
  PHASE3_ENABLE_LIVE_BROKER_ADAPTER=false
  PHASE3_ENABLE_BROKER_RECONCILIATION=false
  PHASE3_ENABLE_STRATEGY_AUTOMATION=false
  PHASE3_ENFORCE_MANUAL_TRADE_CONFIRMATION=true
  ```
- Click "Create Web Service"
- Wait for deployment (~5-10 minutes)

**c) Get Your Render Credentials**
After deployment:
- Note the **Service URL** (e.g., `qlib-api.onrender.com`)
- Dashboard → Copy the **Service ID** from URL (e.g., `srv-xxxxx`)
- Account Settings → API Keys → Create API Key → Copy it

### 3. Add GitHub Secrets

Go to your GitHub repo:
- Settings → Secrets and variables → Actions → "New repository secret"

Add these secrets:

| Secret Name | Value | Where to get |
|---|---|---|
| `VERCEL_TOKEN` | Your Vercel token | Vercel Account Settings → Tokens |
| `VERCEL_ORG_ID` | Your Vercel Team/Org ID | Vercel Account Settings |
| `VERCEL_PROJECT_ID` | Your Vercel Project ID | Vercel Project Settings |
| `RENDER_API_KEY` | Your Render API key | Render Account Settings → API Keys |
| `RENDER_SERVICE_ID` | Your Render Service ID (srv-xxxxx) | Render Dashboard → URL |
| `RENDER_SERVICE_URL` | Your Render Service URL | Render Dashboard (e.g., qlib-api.onrender.com) |

### 4. Update render.yaml CORS

Edit `render.yaml` and update the CORS_ORIGINS line:
```yaml
CORS_ORIGINS=https://qlib-trading-ui.vercel.app
```

### 5. Test Deployment Workflows

Push a test commit:
```bash
git commit --allow-empty -m "Trigger deployment workflows"
git push origin master
```

Monitor the deployments:
- **Frontend**: https://github.com/andybookkeeper/qlib-research/actions → `Deploy Frontend to Vercel`
- **Backend**: https://github.com/andybookkeeper/qlib-research/actions → `Deploy Backend to Render`

### 6. Access Your Live App

Once both workflows pass:
- Frontend: `https://qlib-trading-ui.vercel.app` (or your custom Vercel domain)
- Backend API: `https://qlib-api.onrender.com` (or your custom Render domain)
- Health Check: `https://qlib-api.onrender.com/health`

## Troubleshooting

**Frontend build fails:**
- Check that `src/app/frontend/package.json` exists
- Verify Node.js version in Vercel settings (18+)
- Check build logs in Vercel dashboard

**Backend fails to start:**
- Verify all environment variables are set correctly
- Check Render logs for Python errors
- Ensure `requirements.txt` includes all dependencies

**CORS errors in browser:**
- Make sure `CORS_ORIGINS` in render.yaml matches your Vercel URL
- Restart the Render service after updating
- Check the backend `/health` endpoint for connectivity

**Can't connect frontend to backend:**
- Update vite.config.ts proxy if testing locally:
  ```ts
  proxy: {
    '/api': {
      target: 'http://localhost:8000', // for dev
      // or: target: 'https://qlib-api.onrender.com', // for prod
      changeOrigin: true,
    },
  }
  ```
